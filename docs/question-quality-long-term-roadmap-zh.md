# 拾贝出题质量长期迭代路线图

> 本文档记录出题系统当前最重要的长期问题、根因判断、改进顺序和验收标准。  
> 单轮实验报告回答“这一轮发生了什么”；本文档回答“接下来几轮一定不能忘什么”。

## PRD 验收口径

出题系统的最高验收标准单独沉淀在 [拾贝出题系统 PRD 验收标准](./question-generation-prd-acceptance-zh.md)。

后续每一轮实验都必须先说明它对应哪一条 PRD 验收标准：

- 核心知识点有没有漏。
- 题目是否真的帮助用户理解和回忆。
- 解释和来源是否让用户相信系统。
- 题卡是否足够轻，适合随时复习。
- 评分指标是否能定位真实问题，而不是制造噪音。

如果一轮实验只让题量、低置信比例或某个机器分数变漂亮，但不能解释用户体验为什么更好，不应视为成功。

## 当前阶段判断

截至 2026-05-31，出题系统已经从“题量不足、知识点无题”推进到 **教学质量稳定化阶段**。题量覆盖、来源定位和结构诊断已经基本可用；当前核心问题不是继续增加题，而是确认每道题是否真正完成一个学习动作。

### 2026-05-31 生产基线重设

综合 v4-v13 的实验结果后，当前判断需要修正：这些实验不是一条线性上升曲线。它们确实带来了几个有价值的发现，例如题卡需要保持轻、题目必须有来源支撑、题量不应硬编码、题型不是核心目标；但同时也暴露了一个更大的工程问题：prompt、规则、诊断指标和单篇样本假设不断叠加后，系统开始变重，甚至出现模型注意力被规则抢走、知识点表述变差、过度围绕 Hook 样本定制等副作用。

因此生产生成链路以线上已验证版本的体验为基准重新收缩：

- **保留**：轻量题卡护栏、答案唯一性、来源支撑、动态 1-3 题上限、基础低置信诊断。
- **停用生产强约束**：单篇文章结构绑定、Hook 样本专属结构节点、强制 practiceBlueprint 绑定、为补齐题数而额外 supplement 的生产路径。
- **题量策略**：普通知识点默认 1 道；高价值/高可考知识点 2 道；只有非常关键、证据充足、自然存在多个角度的知识点才到 3 道。
- **prompt 策略**：从“大杂烩规则集”收缩为“可信、轻量、少量高价值”的核心约束，减少模型在无关细则上的注意力消耗。
- **实验策略**：article structure、source block、pedagogical rubric 等复杂诊断继续保留为质量实验工具，不再默认进入生产生成链路。

这次重设不是否定所有实验，而是把实验产物分层：**经验证能改善用户体验的进入生产；尚未证明泛化价值的留在实验室。**

在 `UMr6ia1QubqOMw3aBUGbOw` 单篇基准上，最新已记录结果显示：

- 文章结构已经升级为“主线节点 + 证据子节点”两层。
- v13 最新 run 保留 7 个知识点、21 道题，动态预期题数 21，道题覆盖率为 100%。
- 7 / 7 个知识点达到 3 道题；但只有 6 / 7 个知识点达到完整认知动作覆盖，仍缺 1 个 `scenario_application`。
- 平均每知识点 3.0 道题，题量和动态覆盖已不是当前最大瓶颈。
- 低置信题为 19 道，低置信比例 90.5%；这说明题量达标并不等于教学质量稳定。
- 平均来源精准度 4.8，平均来源最小化 4.7，来源质量仍处高位但略低于 v12。
- 新增低摩擦题卡评分后，平均低摩擦分 4.9、平均可见阅读负担 87.1、高摩擦题 1 道、强制重写级高摩擦题 0 道。说明“轻量复习感”可以被量化并作为护栏。
- 重复练习风险题为 4，道题覆盖恢复的同时仍有换壳练习风险，需要继续校准。
- 主线节点从 v9 的 24 个伪结构节点压缩到 13 个，且关键知识点能绑定到对应主线。

这说明：**题量缺口已经可以补上，题卡轻量感也已建立护栏；但继续加规则并没有带来稳定的质变。** 下一步先验证瘦身后的生产基线，再决定是否把认知动作专项 rewrite 重新引入生产链路。

因此当前最近主线调整为：**Lean Baseline 验证 + 轻量题卡护栏保留 + 实验诊断降级为旁路**。后续不再把“更多题”当作质量提升，也不把全文章固定题数当作硬门槛，而是把“知识点是否准确、题目是否自然可信、来源是否足够支撑、题卡是否适合随手复习”作为第一验收口径。

## 下一阶段：Lean Baseline 验证

v13 证明轻量复习感可以作为评分护栏，但也说明单篇实验继续堆规则会带来反向复杂度。下一阶段先不做 v14 认知动作专项 rewrite，而是验证“瘦身生产基线”是否能恢复线上版本的稳定性，同时保留 v13 已证明有效的轻量题卡护栏。

### Lean Baseline 核心目标

- 不再追求每个知识点都 3 题，动态覆盖率只作为观察指标，不作为唯一目标。
- 平均低摩擦题卡分不低于 4.5，避免质量优化把题卡做重。
- 题目必须保持答案唯一、来源支撑、解释忠实，不用放宽质量检测换数量。
- prompt 长度和生产链路复杂度要下降，避免继续在一个 prompt 里塞满实验规则。
- 验收重点是：知识点表述是否准确、题目是否自然、题卡是否轻、来源是否可信，而不是单篇指标是否更漂亮。

### Lean Baseline 优先修复项

1. **收缩 prompt**：删除生产 prompt 中的单篇结构绑定、过细认知动作流程和补题细则，只保留核心质量规则。
2. **收缩题量目标**：普通点 1 道，高价值点 2 道，极高价值且多角度点 3 道。
3. **停用强补题**：不再为了达到目标题数额外请求 supplement；生成与一次重写后仍不足时接受少题。
4. **分离实验与生产**：articleStructure/sourceBlocks/practiceBlueprint 继续可用于实验报告，但不默认改写生产知识点和题目生成输入。
5. **保留轻量护栏**：题干和选项过长仍触发 rewrite，防止随手复习变成重阅读。

## 已完成阶段：v13 轻量复习感与低摩擦判断

v13 在出题 prompt 和评分系统中加入低摩擦题卡约束：

- 题干只承载必要判断条件。
- 选项是判断对象，不写成解释段落。
- 场景题使用“微场景”，只保留一个角色、一个冲突或一个决策点。
- 复杂背景、原文证据和完整解释放到答后解释页和来源页。

评分系统新增：

- `reviewFrictionScore`
- `visibleReadingLoad`
- `stemLength`
- `maxOptionLength`
- `reviewFrictionReasons`

本轮结果：

- 平均低摩擦题卡分 4.9。
- 平均可见阅读负担 87.1。
- 高摩擦题 1 道。
- 强制重写级高摩擦题 0 道。

结论：**轻量复习感已经可以被量化并作为护栏保留，但它不是当前出题质量的主要瓶颈。** 后续应继续保留低摩擦评分，同时把主要工程投入转向认知动作专项 rewrite。

## 上一阶段：v11-v12 动态题量与认知动作闭环

### 动态题量目标

- 题量由知识点本身动态决定，而不是全文章硬编码一个总数。
- 高价值、高可考、证据充足的知识点目标是 2-3 道，尽量覆盖核心理解、边界辨析、场景迁移。
- 低可考、证据单一、重要性较低的知识点可以降到 1-2 道，但必须记录降级原因。
- 单篇报告不再用 15 / 18 / 24 这类固定数字判定通过；核心指标改为 `actual / expected` 的动态覆盖率。
- 额外题量只有在提升认知动作覆盖、没有引入重复或低质量题时才算正向改动。
- 补题只用于补缺失认知动作，不用于单纯凑数量。

### 认知动作级生成

- `core_understanding`：训练用户抓住核心主张，不能只是问原文复述。
- `misconception_boundary`：训练用户分清真实误区或边界，干扰项必须体现真实混淆。
- `scenario_application`：训练用户把原文原则迁移到新场景，不能只是把原文换壳。
- 同一知识点内，认知动作覆盖优先于题型多样。题型相同但认知动作不同可以接受；题型不同但考同一判断应降级或淘汰。

### 认知动作级评分

- 核心理解题单独评分：是否抓住核心主张、是否避免字面复述、解释是否回到原文主张。
- 边界辨析题单独评分：是否存在真实混淆对象、干扰项是否体现误区、解释是否说明为什么错。
- 场景迁移题单独评分：场景是否合理迁移、正确答案是否由原文原则推出、是否只是原文换壳。
- 低置信原因应具体落到 `core_claim_too_literal`、`boundary_confusion_not_real`、`scenario_is_restatement` 等可修复问题。

### v11 初步结果

`v11-cognitive-action-rubric-loop` 验证了方向，但需要按动态目标重新解读：

- 保留知识点 7 个，入池题 15 道；这个数字本身不构成通过或失败。
- 重复练习风险题降到 0，说明认知动作和去重约束有效。
- 低置信比例从 v10 fix6 的 90.0% 降到 73.3%，但部分改善来自题量减少。
- 高频问题变得更具体：`boundary_confusion_not_real`、`scenario_is_restatement`、`misconception_not_reflected_in_options`。
- 需要补算动态覆盖率：每个知识点的实际入池题数是否达到其 `targetQuestionCount`，以及是否覆盖了应覆盖的认知动作。

因此下一步不是回退认知动作方向，也不是追一个固定总题数，而是校准动态目标：保留具体诊断和重复风险控制，同时检查哪些知识点没有达到自身应有的认知动作覆盖。补题应围绕“修回缺失的认知动作”，不是重新追求 15 / 18 / 24 这类总量数字。

### v12 最新结果

`v12-dynamic-coverage-recall` 在 v11 基础上补齐了动态覆盖诊断，并尝试召回一批相似但认知动作不同的题：

- 保留知识点 7 个，入池题 18 道，动态预期题数 21，道题覆盖率 85.7%。
- `full_coverage` 4 个知识点，`partial_coverage` 3 个知识点。
- 缺失认知动作集中在 `core_understanding` 和 `scenario_application`。
- 来源精准和最小证据保持高位，说明来源系统基本稳定。
- 低置信比例回升到 88.9%，重复练习风险回升到 6，说明“召回”不能替代“重写”和“教学质量校准”。

v12 的结论是：**动态覆盖率和可恢复候选池已经可观测，下一步应把可恢复题改好，而不是继续放宽入池门槛。**

## 第一性原理

拾贝题目系统不是为了“生成更多题”，而是为了帮助用户把一篇文章真正记进脑子里。

一道题进入复习池，至少要满足四个条件：

1. **知识点值得复习**：它属于文章主线、方法、边界、关键误区或高价值案例。
2. **题目能促发理解**：不是简单复述，而是让用户辨析、迁移或回忆关键判断。
3. **答案唯一且解释忠实**：用户答错后能相信解释，而不是怀疑系统在编。
4. **来源片段精准有用**：解释页能把用户带回原文中最相关的位置，而不是贴一大段泛泛上下文。

后续所有改动都要回到这四条判断，避免只追求指标好看。

## 当前根因排序

### P0：低置信题比例偏高，解释和误区支撑需要人工校准

**现象**

sourceBlocks 之后，题量、来源精准度和来源最小化都变好，但低置信比例一度回升到 76.2%。v7 加入 `practiceBlueprint` 后，低置信比例降到 66.7%，但仍偏高。主要机器原因已经从笼统的 `weak_misconception_support`、`weak_explanation_faithfulness`，拆成 `misconception_not_grounded`、`explanation_overextends_source`、`answer_grounding_weak` 等更可修复的问题。

**根因**

来源定位更严格以后，系统开始暴露另一层问题：题目和答案能被来源支撑，不代表解释、常见误区、干扰项也都被同一证据块充分支撑。

这会造成两类问题：

- 题目本身可复习，但解释里多讲了一步原文没有直接支撑的推论。
- 常见误区是模型“合理想象”出来的，而不是来自原文或真实混淆。
- 干扰项和误区之间没有清楚的来源证据边界。

**改进方向**

- 人工抽查本轮低置信题，确认哪些 low 实际可接受。
- 将解释一致性拆成：答案解释、误区解释、干扰项解释三个层面。
- 若 `misconception_not_grounded` 高频成立，要求常见误区必须来自题干、选项或原文证据，不允许泛泛补写。
- 若 `explanation_overextends_source` 高频成立，收窄解释生成规则，只解释来源中能支持的判断。

**验收标准**

- 低置信题人工 accept + fixable 比例稳定高于 80%。
- `explanation_overextends_source` / `explanation_not_tied_to_answer` 人工 reject 率下降。
- `misconception_not_grounded` 不再成为最高频低置信原因。
- 高置信题的解释忠实人工均分高于 4。

### P1：部分知识点 source block 覆盖仍不够分散

**现象**

sourceBlocks 已经让来源段落复用 Top 降到 2 题，但仍有少数知识点 3 道题只使用 1 个 source block，例如本轮 `kp-2`、`kp-6`。

**根因**

这可能是合理的：原文确实只有一个最能支撑该知识点的证据块。也可能说明 block 选择仍偏窄，没有把同一小节里的不同句子窗口分配给不同题。

**改进方向**

- 对每轮 `sourceBlockCoverageByPoint` 中 `questionCount=3` 且 `sourceBlockCount=1` 的知识点做人工抽查。
- 合理复用时记录 `sourceReuseReason`。
- 不合理复用时继续细化 sentence window 或 evidence role 判定。

**验收标准**

- 同一知识点 3 题至少覆盖 2 个 source block，除非有明确复用原因。
- source block 复用 Top 5 中，单块复用不超过 3 题。

### P1：单知识点内部认知动作和题型多样性不足

**现象**

全章题型分布看起来变丰富，但单个知识点内部仍可能 3 道都是同一题型。v6 中多个知识点已经覆盖不同 `memoryAngle`，但仍存在 3 道全是 `scenario_judgment` 或 3 道全是 `multiple_choice` 的情况。

例如：

- 一个知识点 3 道全是选择题。
- 一个知识点 3 道全是场景判断。
- 一个知识点 3 道全是真假判断。

**根因**

v7 已经把 `memoryAngle` 升级为 `practiceBlueprint` 里的练习目标。选择器开始优先覆盖不同蓝图项，再考虑题型多样。当前剩余问题不再是“有没有标 memoryAngle”，而是人工判断这些题是否真的形成“记住 -> 分清 -> 会用”的递进，而不是只在字段上对齐。

PRD 里的“每知识点 1-3 道题”真正意图不是数量，而是：

- 第一题帮助用户取回核心判断。
- 第二题帮助用户辨析误区或边界。
- 第三题帮助用户迁移到具体场景。

如果 3 道题都是同一题型，但分别完成了核心回忆、边界辨析和场景迁移，可以暂时接受；如果 3 道题只是换壳重复，即使题型不同也不应全部入池。

**改进方向**

- 继续把 `practiceBlueprint` 作为正式认知动作契约：
  - `core_understanding`：核心理解 / 主动回忆。
  - `misconception_boundary`：误区和边界辨析。
  - `scenario_application`：场景迁移。
- 每个知识点最多 3 题时，先覆盖不同认知动作，再考虑题型多样。
- 如果题型无法多样，必须记录 `typeDiversityReason`，说明为什么同题型更自然。
- 每道题记录 `blueprintItemId`、`blueprintGoal`、`memoryAngleFitScore` 和 `blueprintAlignmentScore`。
- 同一知识点内相似题干、相似正确理解、相似来源段落需要去重或降级。

**验收标准**

- 每个 3 题知识点至少覆盖 2 个不同认知动作。
- 高价值知识点优先覆盖 3 个不同认知动作。
- 单知识点 3 道题全部同题型时，必须有 `typeDiversityReason`。
- 人工审查时，3 道题应让用户感觉是在从“记住 -> 分清 -> 会用”递进，而不是重复刷同一判断。

### P1：知识点提取还不是完全文章结构驱动

**现象**

当前已经引入 `articleStructureMap`，并在 v10 中升级为“主线节点 + 证据子节点”。但它还不是完整的“先结构、后知识点”链路，仍可能出现：

- 某些文章主线节点被漏掉。
- 模型提取到了但 sourceQuote / 证据块绑定不稳定。
- 细节性、好出题但不关键的点进入复习池。
- 同一个主线节点被多个知识点重复占用，而另一些主线节点没有知识点。

**根因**

知识点提取仍主要依赖模型一次性产出候选，再由后处理绑定结构。当前结构图能辅助诊断和修正绑定，但还没有变成知识点生成的上游约束。

这会导致候选质量受模型即时判断影响较大。

**改进方向**

- 下一阶段考虑真正两阶段知识点：
  1. 先生成文章结构骨架：主题、主张、论证链、方法步骤、边界、案例。
  2. 人工/规则校准哪些主线节点应成为复习点、哪些只是证据。
  3. 再从结构骨架中选择复习知识点，而不是先提知识点再绑定。
- 对每个知识点记录它在文章结构中的位置：
  - `sectionRole`
  - `argumentPosition`
  - `dependsOn`
  - `supports`
- 保持动态数量，不硬编码每篇 5-8 个，但必须解释为什么保留/过滤。

**验收标准**

- 人工审查“漏掉核心知识点”的比例下降。
- 每个保留知识点都能映射到文章结构节点。
- 每轮报告能分清“未覆盖主线节点”和“未覆盖证据子节点”。
- 被过滤知识点有清楚的 `coverageReason`，而不是只靠低可考性。

### P2：复杂分工/边界类知识点缺少专门出题模板

**现象**

涉及工具分工、边界判断、机制对比的知识点，容易出现：

- 干扰项弱。
- 题目太浅。
- 常见误区支撑弱。
- 场景题问法重复。

例如 Hook / CI / Prompt / CLAUDE.md 的分工类知识点，不适合只用普通选择题模板。

**根因**

当前出题 prompt 有通用题型，但没有为“边界/分工/对比”这类知识点设计专门模板。

这类知识点真正的学习价值在于：

- 什么场景该用 A，不该用 B。
- A 和 B 的责任边界是什么。
- 用户容易把哪两个概念混掉。

**改进方向**

- 为 `structureRole = boundary` 或知识点含分工/对比关系时，优先生成：
  - 场景归因题。
  - 工具选择题。
  - 边界辨析题。
  - 错误方案诊断题。
- 干扰项必须来自真实混淆对象，而不是随便凑选项。
- 解释必须明确说明为什么其它工具不合适。

**验收标准**

- 分工/边界类知识点的 `distractor_quality` 人工均分高于 4。
- 这类题的 fixable/reject 率下降。
- 不再出现“题干太简单，没有思考点”的高频反馈。
