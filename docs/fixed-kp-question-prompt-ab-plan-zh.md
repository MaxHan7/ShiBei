# 题目 Prompt 对比实验计划：Full-chain 主实验 + Fixed-KP 诊断

## 背景

v26 的整体质量明显好于前几轮，但 v27 / v28 的小改动出现题量和覆盖下降。复盘后发现，之前仍然在跑完整生成链路：文章清洗、知识点提取、题目生成、质量筛选都会重新发生。这样无法判断问题到底来自题目 prompt、知识点变化，还是后处理误杀。

后续尝试 fixed-KP 后发现：如果完全脱离原始生成链路，只把 v26 的知识点 JSON 冷启动喂给模型，模型行为会明显改变，几乎每个知识点只生成 1 道题。这说明 fixed-KP 适合做诊断，但不适合作为主实验。

下一阶段主实验回到完整链路：每轮从文章开始完整生成，用不同 prompt 说法做小步对比。Fixed-KP / per-point 只用于解释某次结果为什么少题，不用于直接判断生产 prompt 好坏。

## 实验原则

- Full-chain 为主：每轮从文章链接开始完整生成，贴近真实用户路径。
- Fixed-KP 为辅：只在需要定位“知识点变了还是题目生成变了”时使用。
- 小步改动：每轮只改一句或一个局部。
- 先跑 control：每批实验先跑 full-chain v26 control，确认同一时期的自然波动范围。
- 不覆盖基线：v26 原始产物和黄金样本继续保留。
- 不追题量：题量只是健康指标，重点看题干轻量感、选项组质量、正确理解、误区和来源锚点。
- 不靠单次结果：同一 variant 至少跑 2 次再判断趋势。

## 主实验输入

- 单篇文章：`UMr6ia1QubqOMw3aBUGbOw`
- 文章链接：`https://mp.weixin.qq.com/s/UMr6ia1QubqOMw3aBUGbOw`
- 对照基线：`quality-test-set/results/single-article/UMr6ia1QubqOMw3aBUGbOw/runs/20260604-022023-v26-prd-field-standard-lean-prompt.json`

主实验每次都重新跑完整链路。这样会引入模型随机性和知识点波动，但它更接近真实产品路径。为了避免误判，每个关键 variant 至少跑 2 次，并记录每一层数量变化。

## 诊断实验输入

如果 full-chain 某轮题量明显下降，再运行诊断实验：

- Fixed-KP batch：使用 v26 run 的 `chapter.knowledgePoints`，一次性输入所有知识点。
- Fixed-KP per-point：使用 v26 run 的单个知识点，每次只让模型围绕一个知识点出题。

诊断实验只用于定位原因，不直接作为生产 prompt 决策依据。

## 版本设计原则

每个版本必须先写清楚：

- **保留什么**：v26 中已经证明有效的核心原则不动。
- **删除什么**：如果要删 prompt，必须明确删哪一句。
- **新增什么**：新增内容最多 1-2 句，且必须来自字段标准核心问题。

执行顺序不是一次性跑完所有版本。先跑 `A. v26 Control`，再只跑一个最小改动版本。只有这个版本没有退化，才进入下一个版本。

## 第一批 Full-chain 实验矩阵

### A. v26 Control

完整链路使用 v26 当前 prompt，不做任何改动。

目的：确认当前环境下 v26 的自然波动范围。如果 v26 control 自身和原始 v26 差距过大，先不要改 prompt，先判断是模型随机波动、抓取正文变化，还是上游知识点提取变化。

### B. Stem Soft Cap Wording 1

只增加一句：

> 题干能短就短；复杂场景也尽量控制在约 60 字以内。

不删除 v26 任何句子。不加字数下限，不写“必须”，不改变题目质量标准。

### C. Stem Soft Cap Wording 2

如果 B 导致题量下降或题目变浅，换一种更弱的说法：

> 题干优先简洁自然，只有必要时才补充场景背景。

不写数字，不提 60 字。

目的：判断是否是“数字限制”让模型变保守。

### D. Option Length Balance Wording 1

只增加一句：

> 正确项和干扰项长度尽量接近，不要让正确项明显更长或显得更完整。

不删除 v26 任何句子。不要求所有选项等长，不改干扰项标准。

### E. Option Length Balance Wording 2

如果 D 导致选项变得同义或题量下降，换一种更弱的说法：

> 不要让正确项因为信息量明显多于其它选项而暴露答案。

目的：只解决“正确项显眼”，不要求所有选项长度接近。

### F. Two-Line Minimal Repair

只有当题干和选项两个单独版本分别都没有明显退化时，才组合两句。

### G. Delete Nothing Control

作为原则：第一批不删除 v26 任何句子。先只测试新增提示的不同说法。删除 prompt 只在新增说法都失败、且人工审查明确指出某句原 prompt 造成负担时再考虑。

## 后备版本

后备版本不作为第一批实验。只有当前一批指标或人工标注明确指向对应问题时才启动。

- `Remove Boundary Over-Spec`：删除可能让题干显式化过强的边界题约束。
- `Remove Source Over-Spec`：删除可能让模型保守少出题的来源不足少出题约束。
- `Field Responsibility Trim`：压缩字段说明中可能过细的解释。
- `Delete-Then-Add Pair`：用一删一加替换某条轻量题卡原则。

## 关键指标

### 分层生成健康

- 保留知识点数
- 目标题数总和
- 模型实际生成题数
- 评估题数
- 入池题数
- 阻断题数与阻断原因
- 每知识点目标 / 候选 / 入池数量

这组指标用来判断少题发生在哪一层：

- 如果知识点数或目标题数下降：问题在知识点提取/目标策略。
- 如果目标题数没降但模型生成题数下降：问题在题目 prompt 或模型生成行为。
- 如果模型生成题数没降但入池下降：问题在质量评估或题目质量。

### 题干轻量感

- 题干字数：median / p75 / max
- 超过约 60 字的题干数量
- 人工字段审查：`pass / minor_issue / major_issue`
- 用户是否能快速切入题目要考的核心知识点

### 正确项显眼程度

- 正确项字数与干扰项平均字数比例
- 正确项是否明显更长、更完整、更专业
- 正确项是否仍然唯一、清楚、忠实原文

### 干扰项组质量

- 是否存在第二正确答案风险
- 是否有至少一个有学习价值的真实混淆
- 是否存在离谱、无关、同义重复的干扰项
- 三个干扰项作为整体是否形成合理判断空间

## 停机规则

任一 variant 出现以下情况，就不继续扩大改动：

- `evaluatedQuestionCount` 比 fixed-control 下降超过 30%。
- `qualifiedQuestionCount` 比 fixed-control 下降超过 30%。
- `answer_not_unique` 或第二正确答案风险明显增加。
- 人工审查发现正确项和干扰项被写成同义或近义。
- 题干变短但问题变浅、变成背诵式提问。

## 当前执行顺序：修正版

1. 保留 fixed-KP 诊断脚本，但不作为主实验。
2. 给 full-chain 实验增加 trace：记录知识点数、目标题数、模型生成题数、评估题数、入池题数。
3. 跑 full-chain `v26_control_current`，确认当前自然波动范围。
4. 跑 full-chain `stem_soft_cap_wording_1`。
5. 如果题量下降，立即对比分层指标，必要时跑 fixed-KP / per-point 诊断。
6. 如果 wording 1 不稳，尝试 wording 2，而不是继续加规则。
7. 只有题干版本稳定后，才开始选项长度版本。

## 2026-06-04 执行记录

已实现 fixed-KP 实验脚本并跑了两次 v26 control：

- `20260604-075909-v26-control-fixed-kp`
- `20260604-080132-v26-control-fixed-kp-r2`

结果：

- 固定知识点数量均为 8，且与 v26 原始 run 完全一致。
- 原始 v26：入池 20 题，评估 23 题。
- fixed-KP control 第一次：入池 7 题，评估 9 题。
- fixed-KP control 第二次：入池 8 题，评估 8 题。

结论：

- control 没有通过稳定性闸门，不能继续跑 `stem_soft_cap_only` 或 `option_length_balance_only`。
- 下降不是因为字数检测或后处理误杀，主要发生在模型初始题目生成阶段：模型在固定知识点输入下几乎每个知识点只生成 1 题。
- 在 control 稳定前，任何 prompt 变体结果都不可用于判断“这句 prompt 是否更好”。

下一步应先修实验设计，而不是继续改 prompt：

1. 主实验回到 full-chain，贴近真实产品路径。
2. Fixed-KP batch 的结果只作为诊断记录，不再作为 prompt A/B 的闸门。
3. 后续每轮 full-chain 都必须保存分层数量，避免再把“知识点变少”和“题目生成变少”混在一起。
