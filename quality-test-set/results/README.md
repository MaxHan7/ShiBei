# 测试结果目录

运行 `npm.cmd run quality:test` 后，脚本会在这里生成带时间戳的 JSON 文件。

每个结果文件包含：

- `summary`：本轮批量生成的总体统计。
- `results`：每个样本的生成状态、章节、知识点、题目和质量评分。
- `reviewRows`：便于人工评分时复制到表格的题目级摘要。

人工评分建议使用：

- `../manual-scoring-template.md`
- `manual-review-template.csv`

## 人工复核重点

优先检查：

1. 正确答案是否能被来源片段支持。
2. 是否只有一个合理正确答案。
3. 题目是否在考理解，而不是原文填空。
4. 干扰项是否合理但明确错误。
5. 解释是否会误导用户。

如果出现严重问题，请在评分表里标记 `severe_issue = yes`，并记录原因。
