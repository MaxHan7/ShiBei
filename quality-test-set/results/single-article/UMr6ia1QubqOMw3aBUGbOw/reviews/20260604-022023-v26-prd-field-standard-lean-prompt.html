关于hook与prompt的区别，以下哪项正确？

Hook的核心定义：确定性节点控制器
Hook是AI agent关键动作前后的控制器，而非另一个提示词。

Aprompt比hook更可靠
Bhook和prompt完全相同
Chook是系统层面的机制，prompt是模型层面的请求正确
Dhook是写prompt的一种方式

正确理解

Hook是确定性机制，prompt是建议性请求，两者本质不同。

常见误区

认为hook只是prompt的另一种写法，忽略了机制与请求的根本区别。

解释

Prompt是请求模型记住并遵循指令，依赖模型自觉；hook是系统自动执行，不依赖模型决策。

对于每次都必须发生的行为，不应只依赖prompt，应使用hook。

A因为prompt只对单次对话有效，无法跨会话复用
B因为prompt中写不下这么多规则
C因为prompt会降低模型性能
D因为prompt只能请求模型记得做，模型可能忘记，而hook是自动强制执行的正确

正确理解

对于确定性、重复性行为，hook比prompt更可靠，因为hook是系统强制而非模型自觉。

常见误区

认为prompt足够覆盖所有重复行为，忽略了prompt的不确定性。

解释

Prompt是请求模型记住并执行，但模型可能忽略或忘记；hook通过系统机制自动执行，确保每次都发生。

每次代码修改后必须自动格式化，以下哪种方式最可靠？

Hook与Prompt的关键区别：请求 vs 机制
对于每次都必须发生的行为，不应只依赖prompt，应使用hook。

A在项目 README 中写明格式化要求
B在系统提示中要求“每次改完代码记得格式化”
C在 PostToolUse 钩子中设置自动运行 formatter正确
D依赖模型训练时形成的习惯

正确理解

对于每次必须执行的行为，应使用系统机制（如钩子）而非仅依赖提示，因为提示本质是请求，模型可能忽略。

常见误区

认为在 prompt 中强调“必须”就能让模型始终遵守，忽略了模型可能遗忘或忽略提示。

解释

钩子（hook）由系统强制执行，不依赖模型自觉性，能确保每次编辑后都自动格式化。而提示（prompt）只是请求，无法保证。

来源片段

hook 管的是“事件发生时必须执行的动作”。比如改完文件就格式化，跑危险命令前先检查，session 结束前确认测试。它适合确定性、可自动化、可阻断的规则。
