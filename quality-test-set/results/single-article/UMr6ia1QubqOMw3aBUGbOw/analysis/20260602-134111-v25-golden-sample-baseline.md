### 问题本质

这三轮标注说明，自动系统的问题不是单纯“题量不足”或“机器低置信太高”，而是题目字段之间没有稳定服务同一个学习目标。

题干层面，模型经常默认写成 `根据文章`、`文章指出`、`文章最后强调`，这会把产品体验拉回阅读理解测试。拾贝的目标不是检查用户是否记住原文句子，而是帮助用户在手机上快速判断自己是否理解了一个知识点。

误区层面，模型会补一个看起来像误区的句子，但它未必来自原文边界、错误选项或真实用户混淆。例如“产品经理技术不足”这种归因会偏离文章核心；“vibe coding 也注重规范化”甚至把意思写反。误区如果不能帮助用户理解为什么错，就会变成解释页里的噪音。

干扰项层面，模型有两个相反风险：一种是 `放弃使用 AI`、`责怪工程师` 这种太离谱的错误项，用户不用理解也能排除；另一种是把干扰项写得太接近正确答案，破坏答案唯一性。真正有价值的干扰项应该是同语境、同层级、真实可能犯错，但仍然能被知识点边界排除。

同知识点多题层面，q6 和 q7 的连续标注显示，仅仅围绕同一知识点多出题不够。多题必须分工：一题问原因，一题问边界，一题问升级时机或场景应用。否则多题只是在重复考同一个判断。

### 每轮修改思路

第一轮的修改重点是把明显不符合产品心智的内容改掉：去掉模板腔，重写误区，替换低价值干扰项，并补上没有题的知识点。这一轮的价值是建立字段级黄金样本标准：题干、选项、误区、解释都需要可解释，而不是只看题目是否有正确答案。

第二轮的修改重点是纠正“修订候选稿里的新错误”。例如题干压缩不能只是去掉几个字，还要让问题从背诵文章变成判断边界；干扰项不能为了真实而贴近正确答案；正确理解必须写清楚判断对象。它证明黄金样本流程不能只跑一轮，需要用户继续审候选稿。

第三轮的修改重点是精修而不是大改。q5、q6 只换更有价值的干扰项；q8 采纳用户的具体改写；q7 则从重复的“为什么忽略 Hook”改成“什么时候该升级到 Hook”。这轮说明方向已经收敛：后续系统最该学会的是题目分工和真实干扰项，而不是继续增加复杂诊断指标。

### 对后续迭代的参考价值

黄金样本给出了一组比机器指标更可靠的设计约束：

1. 题干要像真实复习判断，不要像阅读理解考题。
2. 每道题只承担一个明确学习任务：原因、边界、升级时机或场景应用。
3. 干扰项必须来自真实错误做法或相邻概念，但不能抢占正确答案。
4. 常见误区必须能映射到错误选项或原文边界。
5. 同知识点多题必须有任务差异；没有差异就不要为题量硬出。

下一轮系统优化应从这些约束里抽出最小 prompt / 规则组合，而不是继续用低置信比例、source coverage 数字或题量做主要方向盘。
