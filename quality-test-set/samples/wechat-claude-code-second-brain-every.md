"当你只是在思考时，必须明确告诉 Claude Code：我不想让你现在帮我写东西。"

"我可以坐在手机上工作两个小时，做真正的思考和研究。"

"如果一个工具能让 10 岁孩子做出一个应用，我看不到它会是泡沫。"

Noah 早年共同创办 Percolate，后来做 Variance，现在经营 AI 策略咨询公司 Alephic，也做一个关于 AI 与营销的会议 brand.ai。本期节目他把Claude code带到现场：Obsidian 笔记库、Git、家里地下室的小服务器、手机终端、客户项目和写作资料，全都接在一起。Dan 说这是他见过最酷的 Claude Code setup，原因不在炫技，核心是每个环节都能落到具体工作：想演讲、查旧资料、修网站、整理 PDF、给客户项目补一个小改动。这期对话真正适合工程师、产品经理和内容工作者看的地方，是 Noah 把“第二大脑”从一个收藏系统，改造成了一个会追问、会整理、会帮你推 PR 的工作伙伴。

Claude Code 先接管的是笔记库

Noah 以前重度使用 Evernote，后来转到 Obsidian。原因很工程师：Obsidian 的笔记就是一堆 Markdown 文件和文件夹，可以同步到 Git，也能被命令行工具直接读写。于是，他把 Claude Code 跑在整个 Obsidian vault 的根目录，而不是只打开某个单独项目。这样一来，Claude Code 可以读他的旧笔记、会议资料、网页剪藏、研究文档，再把相关素材拉进当前项目。

"我使用 Claude Code 的第一大场景，就是让它和我的笔记互动。"

他正在准备一场关于 AI 和营销的演讲，项目文件夹里有 chats、daily progress、research 等目录。外部聊天用 Obsidian Web Clipper 存进来，文章和 PDF 放到 research，每天结束时再让 Claude Code 看当天新增笔记，写出当天学到了什么、哪些想法能推进演讲。项目刚开始时，他还会让 Claude Code 先看过去几场 brand.ai 演讲，理解自己的表达风格，再去全库里找和新主题有关的旧材料。第二大脑在这里承担的任务，是把全库上下文带进当下的思考现场。

想清楚之前，先禁止它写稿

Noah 最有参考价值的习惯，是给 AI 划分“thinking mode”和“writing mode”。他发现模型总想立刻帮你产出最终稿，尤其是当你把演讲、文章、结论这些词丢给它时，它会很快滑向成品。Noah 会在项目说明里直接写清楚：现在不要帮我写任何东西，只帮我思考，问我问题，记录我新给出的想法。

"当你只是在思考时，必须非常明确地说：我只想让你帮我思考，问我问题。"

这件事对今天用 Claude Code 的人很现实。很多人失败在第一步：让 AI 太早生成 artifact，然后自己开始改一份并不成熟的稿子。Noah 的流程更像产品早期探索。他先把过去几场 brand.ai 演讲给 Claude Code，看风格；再给它一个粗标题，比如“Transformers are eating the world”；然后让它从 1500 多条 Obsidian 笔记里找可能有用的材料。等到他卡在结尾时，Claude Code 会围绕他过去几年服务 Amazon、Meta、PayPal 等大公司的经验，帮他整理哪些观察可以放到演讲尾声。先把素材、判断和问题堆到同一个工作台上，成品反而更晚出现。

手机第一次变成深度工作入口

Noah 这套系统最离谱的部分，是他把它搬到了手机上。地下室里一台 mini PC 跑着 Tailscale，Obsidian vault 通过 GitHub 私有仓库同步。他在手机里打开 Termius，连进自己的个人 VPN，再启动 Claude Code。于是，他可以在手机上问“过去两天有什么新东西”，调用自己的 agent，打开别的 repo，甚至修一个会议网站里的坏链接。

"周二我送孩子上校车，然后坐下来吃早餐，真的在手机上做了两个小时的思考和研究。"

手机过去适合处理消息、刷资料、临时搜索，却很少适合写作、研究和改代码。Noah 的变化在于，手机不再负责显示完整工作环境，而是变成远程进入工作环境的窄入口。Claude Code 在服务器上读文件、改文件、跑命令，手机只是一个对话窗口。有一次客户项目快交付，只有一个小改动最适合他来处理；他坐在池塘边，用手机拉下 repo，让 Claude Code 先确认问题，再推了一个解决方案和 PR。另一条线是语音。他开车送女儿去新罕布什尔州夏令营，路上用 Grok voice mode 花了两个小时研究一篇文章；后来 Tesla 也把 Grok 放进车里，他能在路上继续追问 Walter Benjamin、技术精英主义和自注意力。当“工作台”被放到远端，手机终于可以承载一些真正需要上下文的任务。

AI 会钻进组织缝隙里

Noah 准备演讲时研究了 OSS 的《Simple Sabotage Field Manual》。那本二战时期的手册里，白领破坏工作的办法很简单：把事情交给委员会，反复重提已经决定过的问题，在别人要行动时提醒他们不要太仓促。Noah 想借这个材料讲大型组织里的官僚流程，以及 AI 可能如何绕开那些慢动作。

"AI 有点像模糊接口，它不太在意输入输出的形式。"

他给了一个很形象的说法：AI 像 Thomas' English muffin，会钻进 nooks and crannies，也就是组织的缝隙。企业软件过去最难的是 adoption 和 change management，每个人都要迁移到同一个系统、同一种流程。模型作为中间层后，员工可以保留原本的文件、仓库和工具，AI 负责翻译、连接和补齐。Noah 在给大品牌做 AI 项目时，看到的常见难点正是旧系统、旧流程和新需求之间的摩擦；Claude Code 这类工具的价值，往往出现在那些没人愿意重做平台、但又急需把信息接起来的地方。工作方式不一定先被统一，信息可以先被模型打通。

他准备这场演讲时，还把 OSS 手册印了 300 本，自己写了新序言，准备在纽约的 brand.ai 会议上发给参会者。这个细节很能说明他的工作方式：先把公共领域材料、历史人物 Wild Bill Donovan、AI 营销项目和自己的客户经验放在同一个文件夹里，再让 Claude Code 在这些材料之间找连接，最后回到具体演讲现场。

代码复用开始绕过“抽库”

Dan 在 Every 内部也看到类似变化。Every 只有 15 个人，却跑着 6 个产品，每个 GM 可以选择自己的技术栈。Sparkle 做文件组织和 Spotlight 式搜索，另一个产品 Paralegal 用 Claude Code 坐在法律文件目录上，回答“有没有签过这份合同”“员工协议模板在哪里”这类问题。两个产品都需要快速文件搜索。

"我们没有把 Sparkle 的文件搜索抽象成一个所有人都能用的模块，而是让开发者去问 Claude Code：它是怎么做的，然后做自己的版本。"

过去要复用这套能力，团队得抽一个库，统一平台、接口和依赖。现在的做法更轻：把 Paralegal 的开发者加到 Sparkle repo，让 Claude Code 读实现，再迁移出适合自己栈的版本。Dan 把它叫作 tacit code sharing，隐性代码共享。Noah 也在客户和内部工具里这样做，用 GitHub MCP 让 Claude Code 去别的 repo 看最佳实践，再搬到当前项目。比如他们内部有个叫 intelligence 的工具，包着 CRM 和公司运营所需的各种功能；以后新项目遇到类似问题，Claude Code 可以直接去那里找实现痕迹。知识复用从“封装给人调用”，变成“让模型读懂再转译”。

写作、代码和表达会重新混在一起

Noah 谈到自己的 10 岁女儿。家里每年圣诞会做抽礼物对象的活动，她想当那个“chooser”。Noah 把手机和 v0 给她，让她自己 vibe code 一个应用。她最后迭代了 75 次，还在过程中自己碰到数据建模：原来规则写“成人给成人送、孩子给孩子送”，后来她想做成更通用的应用，于是意识到应该叫 groups。

"她开始进入很有意思的计算机科学概念，虽然她自己并不知道。"

Noah 对“vibe coding 泡沫”的回应很直接：他不关心某家公司估值是否过高。一个工具如果能让 10 岁孩子把想法做成可运行应用，它打开的表达空间就很难被一句泡沫盖过去。他还在给 NYU 提一门课，暂名叫 Code is Essay。代码过去长期只属于会编程的人，现在开始变成一种表达媒介，像写作、演讲、剪辑一样，被更多人拿来说明自己的想法。女儿做抽礼物应用时，最有价值的部分并非“写出代码”四个字；她在和工具来回 75 轮里，开始理解规则、对象、分组和通用性。

学校要教会孩子判断机器

教育部分里，Noah 的观点比“禁用 AI”更温和，也更难。他在洛杉矶和一位 11 年级英文老师聊过，对方问学生都用 AI 写作该怎么办。Noah 说，他不知道英文老师的工作该怎么做，但从非常基础的层面看，老师的任务可能不只是教孩子写作，因为写作是一生的事；更重要的是让孩子相信写作值得学。

"你无法把一种不会被藏起来的技术藏起来。"

他把 AI 幻觉和电视、互联网、社交媒体里的错误信息放在同一类能力训练中：学生需要媒体素养，需要知道自己什么时候被一个看似顺耳的答案骗了。Noah 推荐 Tim Harford 写给孩子的《The Truth Detective》，书里有个说法叫 brain guard：当一条信息让你感觉特别好、特别符合自己观点时，反而要更怀疑一点。他还提到 NYU Gallatin 的毕业答辩：学生要在一个房间里，围绕 25 本书和几位教授连续讲清楚自己的学习路径。无论准备时有没有 AI，进房间后都得真的能说清楚。AI 时代的基础能力，不只是会提问，还包括能识别自己为什么轻易相信。

命令行会变成个人操作系统

对 Noah 来说，Claude Code 已经把他重新带回命令行。他最近在 Linux 上玩 DHH 的 Omarchy，不熟悉命令和配置，就给这个机器建了一个专门的 Claude Code 项目，负责帮他改 Neovim、装插件、理解 Linux 操作。他还在 Mac 上做了类似助手，让它清理 Homebrew、把 Python 包管理从 pip 换到 uv，并记住他的偏好设置。

"Claude Code 已经成了我生活里非常核心的一部分，所以我对命令行越来越感兴趣。"

最后一个小工具更像日常刚需：Obsidian 的 attachments 文件夹里，PDF 和图片常常名字很乱。Noah 做了一个 Claude Code 小工具，遍历附件，用 Gemini Flash 重命名，生成元数据表，再把笔记里的附件链接全部改回去。它不像一个宏大的 AI 产品，却非常像未来个人软件的样子：你说出一件重复、琐碎、需要理解文件关系的事，系统直接在你的资料库里完成。Noah 说自己现在“玩得很开心”，白天给 Amazon、Meta、PayPal 这类公司做 AI 项目，业余继续在命令行里搭自己的小系统；这股兴奋感，来自工具终于能进入日常细节。

写在最后

Noah 的做法不适合照抄成一套教程。更值得拿走的是他的判断：Claude Code 的入口不只在代码仓库，也可以在笔记库、演讲项目、手机终端和个人服务器里。先让它贴近你已经在做的工作，再慢慢把重复劳动和上下文检索交出去。工程师可以从一个 repo 开始，产品经理可以从一组 PRD 和用户访谈开始，内容工作者可以从自己的笔记库开始。第二大脑不需要一开始就宏大，先让它帮你守住一个真实流程。
