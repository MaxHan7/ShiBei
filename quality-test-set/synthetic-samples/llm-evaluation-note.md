评估大模型能力时，不应该只看一次问答的惊艳程度。更可靠的做法是先定义任务边界，再准备一组稳定样本，观察模型在同类任务中的平均表现和失败模式。

如果只挑选模型回答得最好的案例，很容易高估它在真实流程中的稳定性。真实业务更关心的是模型在重复任务中是否可控，是否能解释错误，是否能在输入质量变差时保持基本表现。

评估时至少要区分三类指标：第一是正确性，即结果是否符合事实和任务目标；第二是一致性，即相似输入是否得到稳定输出；第三是可恢复性，即模型出错后能否通过提示、约束或人工复核降低风险。

对于高风险任务，模型评估不应只依赖自动评分。自动评分可以帮助筛选明显失败的结果，但最终仍需要人工抽样检查来源支撑、边界条件和潜在误导。
